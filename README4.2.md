## Описание реализации  

**Тема:** Наследование, Полиморфизм и Инкапсуляция       
**Цель работы:** освоить механизм наследования для создания иерархии классов, реализовать полиморфизм для гибкой бизнес-логики и научиться управлять взаимодействием объектов через композицию

**Вариант №3 — Управление активами**
В ходе выполнения работы были разработаны два дочерних класса для Asset:
* `Hardware` - оборудвание (Атрибуты: serial_number - Серийный номер оборудования)
* `SoftwareLicense` - лицензия на программное обспечение (Атрибуты: expiration_date - Дата окончания лицензии)

И один дочерний класс для Employee:
* `ITManager` - IT менеджер (Атрибуты: budget - Бюджет на активы)

В конструкторах дочерних классов были использованы super().__init__(...) для инициализации общих полей (asset_id, name, purchase_price для дочерних классов БК1 и employee_id, name, department для дочерних классов БК2) и инициализированы специфичные поля.

В качестве полиморфного метода был реализован метод calculate_depreciation(), высчитывающий амортизацию для оборудования и софта с разными процентами    

Был реализован класс Контейнер Assignment, конструктор которого принимает объект Работника и создавать пустой список для
объектов Оборудования. Также был реализован метод add_item(item), который добавляет объекты в список. В классе-контейнере был реализован метод calculate_total_depreciation, который перебирает список добавленных объектов, вызывает у каждого объекта его полиморфный метод и в зависимости от класса оборудования, использует нужный процент при рассчете амортизации.

В демонстрационной части программы созданы экземпляры дочерних классов оборудование, лицензия и Айти менеджер; создан набор объектов оборудования; контейнер, в которые они были добавлены и выполнен вывод оформленной в отчет информации о них в консоль.
